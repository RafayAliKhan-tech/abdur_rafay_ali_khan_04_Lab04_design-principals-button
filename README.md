# HCI Principles Demo (Interactive Button)

## Description
This project demonstrates key **Human-Computer Interaction (HCI) principles** using a simple interactive button. The button provides real-time feedback, visual cues, and enforces constraints to guide user interaction.

The demo showcases:
- **Affordance** – The button visually invites clicking with a blue border.
- **Mapping** – The status display shows results clearly with a green border.
- **Feedback** – Clicks highlight the button in yellow.
- **Constraint** – Limits user action after 3 clicks with a red border.
- **Signifier** – Disabled button visually shows it cannot be clicked (purple background).

---

## How It Works
1. Click the **Process Data** button.  
2. Each click updates the status display with the number of times clicked.  
3. A yellow feedback effect briefly appears on the button when clicked.  
4. After 3 clicks:
   - The button is **disabled**.  
   - Borders turn red and the background becomes purple (signifying it’s inactive).  
   - The status shows: ⛔ Limit reached (3 clicks max).

---

## Features
- ✅ Interactive button with click feedback  
- ✅ Real-time status updates  
- ✅ Visual cues for HCI principles: affordance, mapping, feedback, constraint, signifier  
- ✅ Click limit enforcement

## Usage
1. Open `index.html` in any modern web browser.  
2. Click the button and observe the HCI principles in action.  
3. Try clicking more than 3 times to see the constraint and signifier applied.
